被美国遣返、禁止入境 5 年，他在温哥华的卧室里做出了日处理 400 万工作流的 AI 平台

背景：Max Brodeur-Urbas 是 Gumloop 的联合创始人兼 CEO。这家 AI 自动化平台刚拿了 Benchmark 领投的 5000 万美元 B 轮融资，客户包括 Instacart、Shopify、DoorDash、Gusto。但在此之前，他是微软的一名普通工程师，因为过边境被遣返而被迫走上创业之路。

核心观点：

  • “50 个 AI Agent 帮我运营公司”是营销骗局。绝大多数声称全自动化的创业故事都在撒谎，他们真正的赚钱方式是卖课本身。
  • 大厂经历是一种”自我安慰”，大多数人说着说着就被金手铐套住了。21 到 23 岁的时间窗口极其稀缺——没有房贷、家庭负担，应该用来尝试创业而不是每天修 ticket。
  • 从 AutoGPT 的 Discord 服务器中发现了真正机会：用户不是要自主 Agent，而是可靠性和可预测性。Gumloop 从一个简单的 UI 演变成了日处理 400 万工作流的自动化平台。

精彩摘录：

“希望是最好卖的东西。”
“AI 加持，而非 AI 替代。最好的用户是用 AI 增强自己工作能力的人，不是试图用 AI 替掉整个岗位的人。”
“在创业里，你应该追着’证明自己是错的’跑，而不是等着’证明自己是对的’。”
“只自动化自己真正理解的东西。如果你在自动化一个你不理解的流程，你做出来的就是一个老虎机，结果完全不可预测。”
“最后一代伟大的工程师可能已经出生了。”

个人评论：这篇文章最打动我的是 Max 对 AI 行业营销泡沫的清醒认知，以及他从失败中学习、从用户反馈中迭代的真实经历。他对”AI 加持而非替代”的定位，与很多过度宣传”全自动”的公司形成了鲜明对比。

构建 Claude Code 的经验：我们如何使用 Skills

背景：Thariq Shihipar 是 Anthropic Claude Code 团队工程师，也是 Skills 功能的核心推动者。这篇文章是 Anthropic 内部团队使用 Skills 的实战总结——他们内部活跃使用的 Skills 已经有几百个。

核心观点：

  • Skills 不只是 markdown 文件，它们是文件夹，可以包含脚本、资源文件、数据等等。最有意思的 Skills 往往创造性地利用了配置选项和文件夹结构。
  • Skills 大致分为 9 类：库与 API 参考、产品验证、数据获取与分析、业务流程与团队自动化、代码脚手架与模板、代码质量与审查、CI/CD 与部署、运维手册、基础设施运维。
  • 编写好 Skill 的关键：不要说显而易见的事，建一个踩坑点章节，利用文件系统与渐进式披露，不要把 Claude 限制得太死。

精彩摘录：

“任何 Skill 中信息量最大的部分就是踩坑点章节。这些章节应该根据 Claude 在使用你的 Skill 时遇到的常见失败点逐步积累起来。”
“description 字段不是摘要——它描述的是何时该触发这个 Skill。好的 description 读起来更像 if-then 条件，而不是功能说明。”
“Skills 最大的好处之一就是你可以把它们分享给团队的其他人。”

个人评论：这篇文章的价值在于它是来自 Anthropic 内部团队的真实实践，而不是理论探讨。9 种 Skills 类型的分类体系和编写技巧对正在构建或使用 AI 编程工具的团队很有参考价值。特别是”踩坑点章节”和”渐进式披露”的理念，体现了从实战中提炼的智慧。
