1. 编程 Agent 如何重塑工程、产品和设计

来源: 宝玉的分享
发布时间: 2026年3月11日
链接: 编程 Agent 如何重塑工程、产品和设计

摘要:
编程 Agent 让写代码变得异常简单，正在重塑 EPD（工程、产品、设计）的角色定位。传统软件开发流程中，PRD（产品需求文档）曾是核心枢纽，连接产品、设计和工程。但现在，编程 Agent 能把想法直接变成可运行的软件，传统的从 PRD 开始的流程已经终结。

不过，描述产品需求的文档依然不可或缺，只是在交付评审前，这份文档应该是原型的必备搭档。现在的瓶颈已经从实现转向评审——确保代码”足够好”，包括架构合理性、产品思考到位和设计易用性。

通才比以往更有价值，因为他们能同时搞定产品、设计和工程，省掉沟通开销。使用编程 Agent 已成为必选项，好的 PM 会更有价值，而差的 PM 会变得更糟糕，因为糟糕的产品想法现在可以快速实现，但需要更多人来评审和把关。

2. soluna 外挂 C 模块

来源: 云风的 BLOG
发布时间: 2026年3月11日
链接: soluna 外挂 C 模块

摘要:
soluna 集成了 Lua 虚拟机，默认将 Lua 库静态链接到执行文件中，这导致无法以动态库形式外挂 Lua 的 C 扩展。因为在 Lua 的早期版本中，进程空间有多份 Lua 实现会导致运行期错误——所有的 nil 都指向同一个静态”空”对象，不同副本的比较会出问题。

虽然从 Lua 5.4 开始，这个”空”对象被移入 lua_State 运行期结构，解决了错误链接也能不出错的问题，但作者还是认为在同一进程中置入多份 Lua 实现是不好的。

作者提出了一种新的方案：使用 lua_getextraspace() 的技巧，将所有 Lua C APIs 的引用放在一个临时 Lua 虚拟机的 extraspace 中，然后通过自定义的 extlua_init() 函数导入所有 API，最后将入口函数表复制到当前虚拟机并销毁临时虚拟机，完成外部模块的动态加载。这种方法既支持动态链接也支持静态链接（iOS 场景）。
