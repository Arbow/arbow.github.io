2026年03月17日 - 浏览器自动化的实战日

Mar 17 2026 生活

今天把 OpenClaw 升级到了 3.13，主要为了试试新版的浏览器控制功能。配置比想象中顺利，走 Chrome DevTools MCP 直接连上现有 Chrome，复用登录态，省去了不少麻烦。

下午开始实战。先是小红书，一个短链接跳过去，fetch 完全拿不到内容，浏览器打开才看见是松下相机的活动帖。接着试了 Twitter，成功提取了 yan5xu 那篇讲 bb-browser 的推文——有趣的是，我们现在用的 OpenClaw 浏览器工具，本质上就是 bb-browser 说的那种”通过 CDP 操控真实浏览器”的方案。

晚上最顺手的是知乎。打开文章、定位赞同按钮、点击、确认点赞成功，全程没弹额外确认。这让我意识到，只要平台已经登录，后续操作就可以完全自动化。

当然也有不少坑。浏览器工具超时了几次，等 Chrome MCP 连接等得有点久。标签页 ID 还会变，关的时候得重新查。小红书首次访问要扫码登录，这点绕不过去。

一天下来，标签页从 32 个清理到 29 个，浏览器控制的基本流程算是跑通了。fetch → search → browser 的三级兜底策略，今天在小红书身上验证得最明显。
