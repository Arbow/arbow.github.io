1. 飞书 CLI 开源了，为什么 AI Agent 时代，大家都在做命令行工具？

来源: 宝玉

原文链接: 飞书 CLI 开源了，为什么 AI Agent 时代，大家都在做命令行工具？

推荐指数: ⭐⭐⭐⭐ (4/5) - 理解 AI Agent 接口设计的实用指南

核心概要:

飞书开源了 lark-cli，让 AI Agent 可以直接操作飞书完成发消息、查日历、写文档、建多维表格等任务。文章解释了为什么 AI Agent 时代 CLI（命令行界面）重新成为热门接口选择：CLI 是自描述的，AI 只需要运行 --help 就能了解能力、参数和用法，而 API 需要预先获取文档、搞懂认证方式。

文章还厘清了 CLI、MCP 和技能（Skills）三者的关系：CLI 是实际干活的工具；MCP 是另一种让 AI 操作外部服务的方式，但工具描述常驻上下文窗口；技能则是给 Agent 看的说明书，告诉它什么场景该用什么参数。作者总结了为 AI 设计 CLI 的四大原则：help 文本是最重要的文档、支持 dry-run 作为安全网、错误信息要包含修复指引、返回结构化数据并控制输出量。

关键洞察:

  1. CLI 是自描述的接口：AI 遇到陌生 CLI，运行 --help 就能知道怎么用，而 API 需要预先获取文档和认证信息
  2. CLI 比 GUI 更适合 AI：文本输入输出是 AI 的强项，截图+视觉识别+模拟点击是绕远路
  3. CLI、MCP、技能三者互补：CLI 是手，MCP 是另一种手，技能是肌肉记忆；能访问终端用 CLI 更灵活，不能访问终端用 MCP
  4. dry-run 是 AI 的安全网：AI 会自己做决策，dry-run 让用户能在执行前预览和确认
  5. 错误信息要可行动：告诉 AI 哪个参数出问题、具体错在哪、下一步执行什么命令修复

评价:

  • 优点: 概念厘清清晰，CLI/MCP/技能的三元关系解释到位；设计原则实用，可直接指导实践；飞书和 Google gws 的对比让观点更有说服力
  • 不足: 对权限管理、审计追踪等企业级安全问题的讨论较浅；对 CLI 在 Windows 等非 Unix 环境的适用性未提及
  • 适用场景: 适合正在设计 AI Agent 接口的产品经理和开发者，尤其是需要让 Agent 操作现有企业系统的场景

2. Notion 联合创始人 Simon Last：我从去年夏天起就没写过一行代码了

来源: 宝玉

原文链接: Notion 联合创始人 Simon Last：我从去年夏天起就没写过一行代码了

推荐指数: ⭐⭐⭐⭐⭐ (5/5) - 了解 AI 如何重塑知识工作必读

核心概要:

这篇访谈整理了 Notion 联合创始人 Simon Last 在 No Priors 播客上的分享，讲述了 Notion AI 从 2022 年 GPT-4 初体验到 2026 年 Custom Agent 发布的完整历程。Simon 透露 Notion 的 AI harness（围绕大模型构建的系统层）大约每六个月推倒重写一次，很多公司做了一版就不动是常见错误。

文章最引人注目的部分是 Simon 个人的工作方式转变：他从 2025 年夏天起不再手写代码，而是成为”Agent 管理者”。他的个人纪录是一个编码 Agent 连续运行 13 天不停。Simon 还分享了多个实际案例：邮件分拣 Agent 自动归档 95% 不需要看的邮件、反馈路由 Agent 将 Slack 反馈自动路由到正确团队、Custom Agent 可以在后台自主运行并按计划执行任务。

Notion 的使命也发生了根本转变：从”为人类创造最好的直接做事的工具”变成”为人类创造最好的管理 Agent 来替你做事的工具”。Simon 认为编码 Agent 就是 AGI 的内核，代码是表达确定性逻辑的最好原语。

关键洞察:

  1. AI harness 必须持续重写：模型和技术进步太快，系统必须紧贴当前状态设计，做完一版就不动是常见错误
  2. 编码 Agent 改变一切：用得好产出比人写的更健壮，用得差产出全是垃圾；关键是验证循环的设计，不是”凭感觉编程”
  3. 100 倍工程师是真的：会用工具的人产出上限极大提高，最低门槛没变，但最高门槛极大地提高了
  4. 为 Agent 重新设计 API：Notion 为 Agent 设计了 Markdown 方言（页面）和 SQLite 语法（数据库），因为原有 JSON 格式对 Agent 太冗余
  5. 从做事到管理 Agent：Notion 的使命转变预示了知识工作的未来——人负责管理 Agent，Agent 负责执行

评价:

  • 优点: 一手访谈信息丰富，Simon 的多个实际案例（13 天 Agent、邮件分拣、反馈路由）非常具体；对 AI harness 重写周期的坦诚分享难得；Notion 使命转变的总结有启发性
  • 不足: 对 Custom Agent 自主运行的安全风险（prompt injection）讨论较少；对”模型界的瑞士”定位在模型厂商也想做平台的时代能否维持，分析不够深入
  • 适用场景: 适合所有知识工作者了解 AI 如何重塑工作方式，尤其是产品经理、团队负责人和关注 AI Agent 落地的技术决策者
