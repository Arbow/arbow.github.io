1. Claude Code auto mode: a safer way to skip permissions

来源: Anthropic Engineering
发布时间: 2026年3月25日
链接: Claude Code auto mode: a safer way to skip permissions

摘要:
Claude Code 默认要求用户在执行命令或修改文件前进行批准，这种设计保证了安全性，但也带来了频繁点击”批准”的负担，久而久之会导致审批疲劳。用户通常有两种解决方案：使用内置沙箱来隔离危险操作，或使用 --dangerously-skip-permissions 标志跳过所有权限提示，让 Claude 自由行动——后者在大多数情况下是不安全的。

Auto mode 是 Claude Code 的新模式，通过基于模型的分类器来委托认批决策，在人工审核和无防护之间找到了中间地带。目标是捕获与用户意图不符的危险操作，同时让其他操作无需批准提示即可执行。

该模式使用两层防护：输入层和输出层。输入层的 prompt-injection probe 会在工具输出进入代理上下文之前进行扫描，防止内容劫持。输出层的 transcript classifier（基于 Sonnet 4.6）会在执行前评估每个操作，替代人工审批者。分类器分两个阶段运行：快速单令牌过滤器决定是阻止还是允许，只有在第一个过滤器标记时才进入第二阶段的链式思维推理。

该设计针对四种威胁模型：过度热切行为、诚实错误、提示词注入和对齐错误的模型。通过内部事故日志分析，分类器能够识别范围升级、凭证探索、代理推断参数等危险行为模式。

在性能评估中，分类器在真实过度热切行为数据集上达到了 17% 的假阴性率（漏过危险操作的比例）。这意味着虽然分类器并非完美，但相比完全跳过权限提示，它提供了实质性的安全提升。
