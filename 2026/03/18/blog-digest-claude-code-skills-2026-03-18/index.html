《构建 Claude Code 的经验：我们如何使用 Skills》

📰 来源：宝玉博客（翻译） | 📅 时间：2026-03-18 | ✍️ 作者：Thariq Shihipar（Anthropic 工程师）
⭐⭐⭐⭐⭐ 推荐星级

文章摘要

本文作者 Thariq Shihipar 是 Anthropic Claude Code 团队的核心工程师，也是 Skills 功能的主要推动者之一。文章分享了 Anthropic 内部使用 Claude Code Skills 的实战经验——目前内部活跃使用的 Skills 已有几百个。

文章首先澄清了一个常见误解：Skills 不只是 markdown 文件，而是可以包含脚本、资源、数据的完整文件夹结构，智能体可以发现、探索和使用这些内容。

Skills 的九大类型：

  1. 库与 API 参考 - 帮助正确使用库、CLI 工具或 SDK，包含参考代码和踩坑点
  2. 产品验证 - 描述如何测试或验证代码，常搭配 Playwright、tmux 等工具
  3. 数据获取与分析 - 连接数据和监控体系，包含数据获取库和常用工作流
  4. 业务流程与团队自动化 - 把重复工作流自动化为一条命令
  5. 代码脚手架与模板 - 为特定功能生成框架样板代码
  6. 代码质量与审查 - 执行代码质量标准并辅助代码审查
  7. CI/CD 与部署 - 拉取、推送和部署代码
  8. 运维手册 - 引导多工具排查流程，生成结构化报告
  9. 基础设施运维 - 执行日常维护和运维操作

编写 Skills 的核心技巧：

  • 不要说显而易见的事 - 专注于打破 Claude 常规思维模式的信息
  • 建一个踩坑点章节 - 记录 Claude 使用时的常见失败点
  • 利用文件系统与渐进式披露 - 把整个文件系统当作上下文工程工具
  • 不要把 Claude 限制得太死 - 给信息但留灵活性
  • description 字段是给模型看的 - 描述”何时该触发”而非”做什么”
  • 记忆与数据存储 - 可用 ${CLAUDE_PLUGIN_DATA} 存储数据
  • 存储脚本与生成代码 - 给 Claude 脚本和库，让它专注组合编排

分发 Skills：

  • 提交到代码仓库（./.claude/skills）
  • 搭建内部插件市场（Plugin Marketplace）
  • 让有用的 Skills 自然涌现，通过 PR 审核机制正式发布

核心观点

“Skills 是 AI 智能体极其强大且灵活的工具，但这一切还处于早期阶段，我们都在摸索怎样用好它们。”

文章强调 Skills 的最佳实践是：从小做起（几行文字加一个踩坑点），通过不断迭代、补充 Claude 遇到的新边界情况来完善。

特别值得注意的是 description 字段的写法——很多人写”这个 Skill 做什么”，但 Claude 需要的是”什么情况下该用这个 Skill”。好的 description 读起来更像 if-then 条件。

精彩摘录

关于 Skills 的本质：

“Skills 最有意思的地方恰恰在于它们不只是文本文件——它们是文件夹，可以包含脚本、资源文件、数据等等，智能体可以发现、探索和使用这些内容。”

关于踩坑点章节：

“任何 Skill 中信息量最大的部分就是踩坑点章节。这些章节应该根据 Claude 在使用你的 Skill 时遇到的常见失败点逐步积累起来。”

关于渐进式披露：

“把整个文件系统当作上下文工程（Context Engineering）和渐进式披露（progressive disclosure）的工具。告诉 Claude 你的 Skill 里有哪些文件，它会在合适的时候去读取它们。”

关于按需钩子：

“/careful —— 通过 PreToolUse 匹配器拦截 Bash 中的 rm -rf、DROP TABLE、force-push、kubectl delete。你只在知道自己在操作生产环境时才需要这个。”

为什么推荐

  1. 一手经验 - 来自 Anthropic 内部团队的实战总结，不是理论推演
  2. 体系完整 - 从 Skills 分类、编写技巧到团队分发，覆盖完整链路
  3. 可落地 - 每个技巧都有具体示例，可以直接参考应用
  4. 前瞻性强 - Skills 是 Claude Code 的核心扩展点，代表了 AI 编程助手的发展方向

对于正在使用或准备使用 Claude Code 的团队，这篇文章提供了从”会用 Skills”到”精通 Skills”的完整路径。
