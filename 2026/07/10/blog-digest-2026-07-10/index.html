1. 小孩的英语辅导

来源：https://blog.codingnow.com/2026/07/learning_english.html
作者：云风
日期：2026-07-09

核心概要

云风写了一篇非常个人化的长文，记录了他花半年时间亲自辅导儿子（云豆）学习英语的全过程。文章从孩子小学毕业这个时间节点切入，回顾了从三岁时的语言发育迟缓、疫情期间的网课、多邻国打卡，到五年级英语成绩下滑、最后决定系统补习语法的完整经历。

他选择了《剑桥初级英语语法》（English in Use）作为教材，自己当老师，每天保证半小时的学习时间。文章的亮点不在于英语教学技巧本身，而在于云风对”学习如何学习”这件事的深刻反思：孩子读完一页书却忽略了一半的内容、把陌生知识当成已掌握的、找不到重点——这些都是需要”人坐在旁边引导”的过程。他还特别强调了”坚持不懈”比集中突击更重要这一朴素道理。

文章结尾还提到了给三年级的二女儿可可补习英文的计划，从每天半小时朗读开始。

关键洞察

  • 为什么不请家教而自己教：第三方教学的评价反馈机制不足——老师面对学习能力不足的学生时难以投入足够感情，最终只能依赖”背了多少单词、考试多少分”这些与”真正学会”之间有偏差的客观标准。
  • “不间断”本身起的作用更大：每天哪怕只有半小时，持续五个月的效果，远比每周集中一两次两小时以上的课程更有效率。
  • 阅读是一种独立能力：孩子并不是天生会读书的，”提取重点、预设问题、从文字中寻找答案”需要专门引导。这本语法书同时也是学习”如何自学”的教材。
  • 英语学习不只是英语：将英文句子用中文准确表达，本质上是在练习语言表达能力，而不仅仅是语言转换。

评价

优点：真诚、细致、有温度。这不是一篇说教式的教育指南，而是一个父亲的真实观察和复盘。云风作为资深程序员，将分析问题的思维方法迁移到育儿上，对”学习”本质的思考值得所有家长和终身学习者借鉴。文章写得完全没有 AI 的痕迹，是难得的深度长文。

不足：文章聚焦于个人家庭的具体情况，通用性有限。对于不关心育儿话题的读者，可能难以产生共鸣。英语教学方法也并非创新——使用剑桥经典教材、每天固定时间练习，都是比较传统的路线，没有涉及 AI 辅助语言学习等新技术路径。

适用场景：有学龄期孩子的家长、关注教育方法论的人、云风博客的长期读者。对于正在寻找”如何帮孩子建立自学能力”的家长，这篇文尤其值得一读。

推荐指数：⭐⭐⭐ (3/5)

个人育儿随笔，文笔和思考深度俱佳，但内容主题与博客推荐常见的 AI/技术方向有所偏离，适合特定读者群。如果你想从技术内容中换个口味，看看程序员如何用工程思维思考教育，这篇很有价值。

2. 从零开始玩转循环 (Getting started with loops)

来源：https://baoyu.io/blog/2026-07-06/claudedevs-2074208949205881033
作者：宝玉（译自 Claude Devs 团队分享）
日期：2026-07-06

核心概要

这是宝玉翻译/整理的 Claude Code 团队的一篇分享，主题是”设计循环（designing loops）”——不是简单的提示词，而是让 AI 智能体不断重复执行工作周期，直到满足预设停止条件的系统性方法。文章的核心价值在于：它不是讨论”AI 能做什么”，而是讨论”如何用工程化的方式组织 AI 的工作流程”。

文章将循环分为四种类型：回合制循环（用户输入驱动，适合短任务）、目标导向循环（通过 /goal 定义退出标准，适合可验证任务）、基于时间的循环（/loop 和 /schedule，适合周期性和外部系统对接任务）、主动式循环（事件触发，无需人类实时参与，适合工单分诊、依赖升级等持续工作流）。每种类型都给出了触发方式、停止条件、适用场景和 Token 消耗控制策略。

特别实用的是文章最后关于”保持代码质量”的部分：保持代码库整洁（Claude 会模仿已有模式）、通过 Skills 提供自我验证方法、使用第二个无上下文污染的智能体做代码审查。

关键洞察

  • 验证标准越量化、越具体，Claude 就越容易自我判断：在 SKILL.md 中用可测量的标准（如”Lighthouse 跑分 90 以上”、”启动 dev server 并浏览器截图验证”）替代模糊的”做得好一点”，是让循环高效运转的关键。
  • 不是所有任务都需要复杂循环：文章反复强调”从最简单的方案开始”，回合制循环能解决大部分问题，只有在需要持续迭代或对接外部系统时才升级到更复杂的模式。
  • Token 消耗必须有清晰边界：循环必须设置回合数上限、时间间隔、以及”用更小模型处理简单环节”的策略，否则很容易烧掉大量 token 却没有产出。
  • 长循环中最大的敌人是上下文腐烂：越跑越多的旧工具输出和过时推理会让模型表现持续下降，好的循环需要在获得有意义的观察结果后刷新上下文。

评价

优点：结构清晰，每种循环类型都配有触发方式/停止条件/适用场景/Token 控制的四维分析框架，实操性极强。宝玉的翻译质量很高，术语处理到位（如 “primitive” 译为”基础指令”并加了括号解释），对不熟悉 Claude Code 生态的中文读者也很友好。文中大量具体的命令示例（/goal、/loop、/schedule）可以直接拿来用。

不足：内容本质是对 Claude Code 特定产品功能的介绍，对不使用 Claude Code 的读者参考价值有限。文章更像是产品文档/教程的中文版，原创洞察较少。文中提到的”动态工作流（dynamic workflows）”和”自动模式（auto mode）”仍标注为研究预览版，部分实践建议的通用性有待验证。

适用场景：Claude Code 用户、正在构建 AI Agent 工作流的开发者、团队技术负责人。如果你正在设计一个需要 AI 持续运行的自动化流程（如自动处理 bug 报告、定期代码审查），这篇文章的四分类框架可以直接作为你的设计起点。

推荐指数：⭐⭐⭐⭐ (4/5)

实用的工程指南，四分类框架清晰可操作。扣一分是因为高度绑定 Claude Code 生态，非该生态用户需要自己映射概念。对于正在搭建 AI Agent 自动化流程的团队来说，这是一篇值得收藏的参考文章。
