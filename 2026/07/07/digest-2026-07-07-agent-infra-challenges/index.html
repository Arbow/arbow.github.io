文摘：当 Agent 走向生产，Infra 面临哪些挑战？

Jul 07 2026 文摘

作者: 此间的山林
发布日期: 2026-07-07
链接: 微信公众号原文
推荐指数: ⭐⭐⭐⭐ (4/5) — Agent Infra 系统性思考，从工程现象到第一性原理

核心概要

这是作者在蚂蚁集团「一技半解」分享的文字整理版。文章从 Agent 走向生产的工程现象出发，逐层深挖到第一性原理，核心论点是：干活的主体从人变成 Agent，系统性质从确定走向随机。软件对抗复杂度的使命没变，但战场变了。

文章沿「两个变换 → 复杂度平移 → Infra 三个核心特性 → 价值二分 → 认识论锚点」的主线展开，最终推到验证的终局——问责基础设施（Accountability Infrastructure）。

核心观点

1. 两个变换：所有推导的起点

  • 变换一（主体）：从人到 Agent。不只是换了个用户——Agent 以机器速度和规模干活，自主决策，7×24 不停。
  • 变换二（系统性质）：从确定性到随机性。模型输出非确定，Agent 任务长时运行、有状态。

致命之处在于：今天绝大多数 infra 是为「确定性、人驱动、请求-响应、无状态」设计的。Agent 在每个维度上都打破了这些假设。

2. 复杂度没有消失，它只是平移了

Coding Agent 让写代码成本骤降，复杂度从「写」平移到了构建、测试、验证、部署。交付瓶颈移到了 Infra。

信任被验证取代：对人类同事，信任意图与能力，code review 是抽检；对 Agent，无法信任意图，只能验证行为，必须全量验证，验证量爆炸。

Sonar 2026 调查：96% 开发者不完全信任 AI 生成的代码，却只有 48% 在提交前真正做了验证。Werner Vogels 把这个落差称为 Verification Debt（验证债）。

作者的增量判断：Infra 的新使命，是把验证工业化。

3. 运行底座 ≠ 价值底座

先把 Infra 的位置摆正：在 Agent Loop（Harness → LLM + Knowledge + Infra）里，Infra 像交通设施——脱离它不行，但它不是竞争核心。评判标准只能是业务指标：有没有让任务交付更快、更安全、更便宜。

4. 第一性原理：理想与现实的差就是 Infra 的价值

把「理想 Agent」的假设逐条列出，对上物理现实，Infra 承接的压力自动浮现：

理想假设 物理现实 压力落在
无限知识 知识有截止日期、召回不准 统一存储、检索
无限算力 资源有限、成本有限 弹性调度、按需计算
精准长记忆 上下文窗口有限 记忆管理、状态持久化
不出错的执行 随机输出、可能被注入 隔离、沙箱、边界强制
可预测的世界 开放系统、未来不可知 弹性、容错、ad-hoc 响应

关键结论：Infra 没有出现新原语，变的是架构对它们的要求权重。这一代的权重压在三个特性上：快、隔离、弹性。

5. 三个核心特性

快——有三层：

  • 表层：启动快、资源交付快
  • 第二层：整合统一（Agent 每多跳转一个系统 = 一次延迟 + 一批 Token + 一个出错点）
  • 第三层：开放标准 + 内省（SQL/S3 已固化进模型权重 ≈ 零延迟理解；系统需提供结构化、可预判的反馈）

隔离——控制不可信执行者的爆炸半径：

  • 执行者现在是半可信且随机的，威胁是全新的（Prompt Injection、误删数据、越权访问）
  • 边界必须由 Infra 物理强制，不能指望 System Prompt 的软约束
  • 推到极限就是端侧：隐私和凭据不可上云时，信任边界有了物理位置

「隔离不是限制 Agent，是给它自主行动的底气。」

弹性——扛住预测不了形状和时机的负载：

  • 负载单位脱钩了：以前按人数规划，现在按 Agent 调用数走
  • 查询模式变了：Agent 问的是它当场想到、你从没见过的问题
  • 架构方向：存算分离、按需计算、热点自动物化

6. 三选二的历史正在被打破

快（热的、共享的常驻资源） vs 隔离（全新的、独立的、用完即焚） vs 弹性（scale-to-zero、ephemeral）——历史上只能三选二。Agent 时代逼你三个全要。

这解释了为什么 microVM（Firecracker）、快照恢复、CoW fork（亚毫秒级）、V8 isolates 集体翻红。这不是「不可能三角」，是一条工程成本曲线，可以被技术往下压。

7. 价值二分：补短板 vs 处理现实约束

  • 第一类（过渡性）：补当前模型缺陷，如 RAG。会随模型能力演进动态变化。
  • 第二类（结构性）：来自世界本身——多主体、共享、对抗、不可预测。不随模型变强而消失。

长期价值的锚点，在于处理真实世界的约束。

8. 即使 Agent 全知全能，三件事仍然存在

  1. 协调是分布式的属性，不是智力的属性：两个全知全能的 agent 并发写同一份共享状态，仍需一致性协议。上限是光速，不是智力。
  2. 隔离的价值来自有敌意的世界：Prompt injection 不是 Agent 蠢，是输入带恶意。
  3. ad-hoc 的根是信息限制，不是算力限制：明天用户会问什么 query，再强的模型也不知道。

9. 最深一层：Infra 是模型的认识论锚点

借用哥德尔不完备定理的思路：理性有它的局限，「不可判定」的正确解读是「需要一个外面」。对 Agent 而言，这个「外面」就是 Infra、执行、现实本身。模型的推理可以错，但编译器不撒谎，数据库里的事实不受人类认知约束。

10. 验证的终局：问责基础设施

规模大到一定程度，人必然退出验证循环（看不过来）。验收标准从「绝对正确」退到「错误率低于基线」。陷阱是同源自审：让模型验证模型，盲区相关，最自信的地方往往是系统性错误所在。

人退出后三个硬代价落在 Infra 上：

  • 异源验证：最强的异质性不是换一个 LLM，是接一个非模型的裁判
  • 现实锚点：用编译器、规则、现实反馈压住相关错误
  • 全程可审计：社会接受「AI 犯错」的前提是错误可审计、可追责

可观测性在 Agent 时代升了一格：从运维工具，升格为问责基础设施（Accountability Infrastructure）。

关键金句

「干活的主体从人变成 Agent，系统性质从确定走向随机。软件对抗复杂度的使命没变，但战场变了。」

「隔离不是限制 Agent，是给它自主行动的底气。」

「协调是分布式的属性，不是智力的属性。上限是光速，不是智力。」

「Infra 永远有活干，不是因为模型不够强，而是「强」本身就内含一个它够不到的外面。」

评价

这篇文章是近期中文社区关于 Agent Infra 最有深度的一篇系统性思考。 它的价值在于：

  1. 从现象到原理的完整链路：不是堆砌技术名词，而是从「两个变换」出发，一步步推出 Infra 的新使命，逻辑链清晰且有说服力。

  2. 第一性原理思考：「理想 Agent 与物理现实之间的差就是 Infra 的价值」——这个框架把 Infra 的价值来源做了很好的分类：哪些是过渡性的（补模型短板），哪些是结构性的（处理真实世界约束）。这个区分对从业者做技术选型和投资判断都很有参考价值。

  3. 认识论锚点的升维：把 Infra 从「管道」升维到「模型的认识论锚点」，引用哥德尔不完备定理做类比，虽然有些哲学跳跃，但核心直觉是对的——Agent 需要一个「外面」来逃逸出训练数据的局限。

  4. 问责基础设施的洞察：同源自审的陷阱（让模型验证模型，盲区相关）是一个非常实际的工程问题。异源验证、现实锚点、全程可审计这三个方向，指出了可观测性基础设施的演进路径。

不足之处：

  • 部分图片在微信中无法提取，可能丢失了一些视觉化的论证细节
  • 对「快、隔离、弹性」的具体技术方案讨论偏概括性，实操层面的细节有限
  • 创投图谱部分是趋势信号而非完整统计，参考价值有限

与 Lilian Weng《Harness Engineering》的关联

这篇文章和 Lilian Weng 同期的 Harness Engineering 文章形成有趣的互补：

  • Lilian 关注的是 harness 本身如何被优化（上下文工程、工作流设计、自改进循环）
  • 此间的山林关注的是 harness 跑在什么样的 Infra 上（快、隔离、弹性，以及验证工业化）

两者合在一起，构成了 Agent 生产化的完整技术栈：从模型到 harness 到 infra，每一层都有自己不可替代的价值。
