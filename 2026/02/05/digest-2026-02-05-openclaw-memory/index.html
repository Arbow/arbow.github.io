内容概要

这是一篇关于 OpenClaw Memory 模块的详细设计文档，介绍了 AI Agent 的语义搜索记忆系统架构。

核心架构

Memory 模块采用混合搜索架构，结合向量相似度（cosine similarity）和 BM25 全文检索：

关键技术点

组件 技术 说明
Embedding OpenAI / Gemini / Local 多提供商自动降级
存储 SQLite + sqlite-vec 向量 + FTS5 全文检索
搜索 Hybrid (Vector + BM25) 加权合并结果
同步 File Watcher + Delta 增量索引，实时同步

支持的 Embedding 提供商

  1. OpenAI - text-embedding-3-small（云端高质量）
  2. Gemini - embedding-001（Google AI 替代方案）
  3. Local - embeddinggemma-300M（隐私优先，离线可用）

智能同步策略

  • Memory 文件: 基于文件监听（chokidar）的实时同步
  • Session 对话: 基于 Delta 的增量索引（字节/消息阈值触发）
  • 定时扫描: 每 N 分钟全量检查
  • 按需同步: 搜索时的懒同步（如果数据已变更）

Agent 工具接口

  • memory_search: 语义搜索记忆文件和对话历史
  • memory_get: 安全读取特定行范围的内容

性能优化

  • Embedding 缓存: 基于 SHA256 的去重缓存
  • 批量处理: OpenAI Batch API，可配置并发度
  • 安全重建: 原子文件交换，失败自动回滚
  • 优雅降级: FTS 不可用时回退到纯向量搜索

个人点评

这篇文档详细阐述了 OpenClaw 的记忆系统设计，几个亮点值得关注：

  1. Hybrid Search - 向量 + 全文检索的组合比单一方案效果更好
  2. Local Embedding - 支持本地模型，保护隐私且可离线使用
  3. Delta Sync - 对话历史的增量同步设计很巧妙，避免重复处理
  4. Auto Fallback - 多提供商自动降级保证可用性

对于构建 AI Agent 系统的开发者来说，这是一份很好的参考设计。
