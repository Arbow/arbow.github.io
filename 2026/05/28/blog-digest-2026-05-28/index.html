1. Building Self-Improving Tax Agents with Codex

来源：OpenAI Engineering
日期：2026-05-27
推荐指数：⭐⭐⭐⭐ (4/5) — 推荐，实操性强，展示了 Agent 在真实业务场景中的自改进闭环

核心概要

OpenAI 与 Thrive Holdings 合作，基于 Codex 构建了一套自改进税务 AI 系统。该系统通过 Crete Professional Alliance（30+ 会计事务所网络）进行了实际部署，处理了约 7,000 份税务申报，主要针对 1040 和 1041 表格。

这不是一个简单的”AI 辅助填表”工具。其核心亮点在于自改进循环：系统从执业会计师的反馈中学习——每次人类修正 AI 的错误时，系统会记录完整的修正痕迹（production traces），然后利用 Codex 运行定向评估和代码修改，持续优化自身表现。从实践效果看，上线时只有 25% 的申报能达到 75% 字段正确率，六周后这一比例跃升至 86%。从业者报告准备时间节省约三分之一，吞吐量提升约 50%。

值得注意的是业务层面的安排：OpenAI 于 2025 年 12 月收购了 Thrive Holdings 的股权，双方工程师合作六个月共同开发。而最终产出的知识产权和产品归 Thrive Holdings 所有——这在大型科技公司 AI 合作中并不常见。

关键洞察

  • 自改进闭环是核心差异化：不是一次性训练的模型，而是利用生产环境的修正反馈持续迭代，这与传统 ML 模型的 retraining cycle 有本质区别——更快、更精准、更贴近业务
  • 从中高复杂度场景切入：系统刻意避开了简单 W-2 申报，专攻涉及 K-1 数据和复杂文档的中高难度场景，这正是 AI 能产生最大价值的地方
  • 六周内从 25% 到 86%：自改进循环的效果在短期内即可量化，证明了 Agent 在有监督反馈的生产环境中可以快速超越初始能力
  • 知识产权安排值得关注：Thrive 持有 IP 意味着 OpenAI 在某些合作中愿意让渡产品所有权，这对行业合作模式有参考意义

评价

优点：文章提供了清晰的技术架构描述和可量化的业务成果数据（7,000 份申报、准备时间 -33%、吞吐量 +50%），展示了一个真正的 Agent 生产级应用案例，而非 demo。自改进循环的设计思路对其他垂直领域（法律、医疗、保险）有很强的借鉴价值。

不足：原文对 Codex 具体如何执行代码修改和评估的细节不够深入，缺少对错误处理、合规性验证和安全防护机制的讨论。97% 的准确率数字在传播中被简化，原文并未以这种方式呈现这一单一指标。另外，OpenAI 与 Thrive 的投资关系可能影响报道的客观性。

适用场景：正在构建 AI Agent 应用的工程师和产品经理，特别是关注”Agent 如何在生产环境中持续学习”这一话题的读者。对税务科技和 AI 在专业服务领域落地感兴趣的从业者也会从中获益。
